Supabase OX 퀴즈 — 엑셀(CSV) 가져오기용 양식

[1] quiz_subjects.template.csv  → 테이블 public.quiz_subjects
  id              UUID. 비우면 가져오기 스크립트에서 생성 가능.
  name            과목명 (앱의 과목 선택과 동일해야 함)
  sort_order      과목 나열 순서 (0부터)
  is_active       true / false

[2] quiz_units.template.csv  → 테이블 public.quiz_units
  id                  UUID
  subject_id          quiz_subjects.id 와 일치
  parent_unit_id      대단원이면 비움(null). 소단원이면 부모 대단원의 id.
  name                단원명
  sort_order          같은 부모 안에서 순서 (0부터)
  is_active           true / false

[3] quiz_questions.template.csv  → 테이블 public.quiz_questions
  id              UUID
  unit_id         소단원(또는 단일 단원)의 quiz_units.id — 문제는 항상 leaf 단원에 붙음
  question        지문(선지 번호 제외 본문)
  choice_text     선지 본문 (예: ① … 내용에서 ① 라벨을 뺀 텍스트)
  body            선택. 비우면 앱이 question+choice_text 로 조합 가능. 있으면 `문제: …\n\n선지: …` 형태 권장(data.js 의 body 와 동일)
  answer          OX 정답. true = O(맞음), false = X(틀림)
  explanation     해설 (없으면 비움)
  sort_order      같은 unit_id 안에서 순서 (0부터)
  pack_no         같은 4지선다를 쪼갠 OX 행은 동일한 정수로 묶음. 없으면 비움
  is_active       true / false

  ▶ quiz_questions.template.csv 예시 데이터
    - `data.js` 의 MDP_HISTORY_19G01_STEM(19경간 01·4지선다)과 동일한 4행(OX)이 들어 있습니다.
    - `unit_id` 는 `supabase/seed_haeyang_mdp.sql` 의 소단원「해양경찰의 역사」id(a3418c9f-0641-5cf6-b4fe-39ae5cfb50b0)입니다. 다른 DB면 본인의 leaf unit UUID 로 바꾸세요.
    - `id` 는 seed SQL 과 동일한 예시 UUID 입니다. 신규 삽입 시 비우거나 새 UUID 를 쓰면 됩니다.
    - 지문을 `data.js` 와 다시 맞추려면(UTF-8 BOM 유지):  ox-quiz-app 폴더에서
        py -3 tools/import-templates/_gen_quiz_questions_template.py

[4] quiz_flat_entry.template.csv — 작성 편의용(한 시트에 입력 후, 나중에 3개 테이블로 쪼개는 스크립트를 만들 수 있음)
  subject_name, subject_sort_order
  major_unit_name, major_sort_order   (대단원 없으면 비우고 minor 만 써도 됨 — 단 push 스크립트와 동일한 트리가 되게 맞출 것)
  minor_unit_name, minor_sort_order
  pack_no, question_sort
  question, choice_text, body, answer, explanation, is_active

※ UUID 는 Excel 에서 과학적 표기법으로 깨질 수 있으니, 셀 서식을「텍스트」로 지정한 뒤 붙여 넣거나, 가져오기 시 스크립트에서 v5 등으로 생성하는 방식을 권장합니다.

※ 실제 upsert/삽입은 Service Role 로 API 또는 Supabase Table Editor / SQL 을 사용하세요. data.js 동기화는 npm run db:sync (push-curriculum.mjs) 참고.
